# Contributing to UniCalc

Thanks for your interest in contributing. This project is built and maintained by students, and every contribution matters.

## Ways to contribute

### Merit data
If you have verified closing merits for any university, campus, or program, open a PR or issue with the data. Include the year and source. Merit data lives in `js/meritData.js`.

### Aggregate formulas
If you know the exact aggregate formula or test pattern for a university we have not covered, share it. The formula configs live in `js/universityConfigs.js`.

### Bug fixes and features
Fork the repo, make your changes, and open a pull request.

## Before you submit a PR

- Keep it focused. One fix or feature per PR.
- Test on mobile. Most of our users are on phones.
- Do not add frameworks, build tools, or package managers. This project is intentionally vanilla HTML, CSS, and JS.
- Do not add unnecessary abstractions. Simple and readable code is preferred.
- Make sure nothing is broken by opening the page locally before submitting.

## Running locally

```
npx http-server -p 8080
```

Open `http://localhost:8080` in your browser and test your changes.
